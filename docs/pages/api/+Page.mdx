## Common Patterns

### Configuration

```ts
// vite.config.ts
import { photon } from '@photonjs/core/vite'

export default {
  plugins: [
    photon({
      server: './src/server.ts',
      entries: {
        'api/users': './src/api/users.ts'
      }
    })
  ]
}
```

### Server Setup

```ts
// src/server.ts
import { Hono } from 'hono'
import { apply, serve } from '@photonjs/hono'

const app = new Hono()
apply(app)
export default serve(app)
```

### Universal Middleware

```ts
// Universal middleware works across all frameworks
const middleware = createUniversalMiddleware((context) => {
  // Middleware logic
})

apply(app, [middleware])
```

## Type Definitions

Photon provides comprehensive TypeScript definitions:

```ts
import type { Photon } from '@photonjs/core'

// Configuration types
type Config = Photon.Config
type ConfigResolved = Photon.ConfigResolved

// Entry types
type Entry = Photon.Entry
type EntryServer = Photon.EntryServer
```

## Virtual Modules

Photon uses virtual modules for build-time code generation:

```ts
// Available in your code
import handler from 'photon:server-entry'
import { getMiddlewares } from 'photon:get-middlewares'
```

## Error Handling

Photon provides specific error classes:

```ts
import {
  PhotonError,
  PhotonConfigError,
  PhotonRuntimeError
} from '@photonjs/core'
```

## CLI Commands

When using Photon with Vite:

```bash
# Development
vite dev

# Build
vite build

# Preview
vite preview
```

## Environment Variables

Photon respects standard environment variables:

- `NODE_ENV` - Environment mode
- `PORT` - Server port (development)
- `HOST` - Server hostname (development)
